The feature must work on top of every Home game:
- Karaoke

- WordBack

- Percorso a Risate

==================================================
GOAL
==================================================

Create a lightweight live social messaging system.

Players can:
1. send private messages to another connected player
2. send anonymous private messages
3. send signed private messages
4. send messages to TV queue
5. react to received messages
6. enable/disable Do Not Disturb

This is NOT a full chat thread.
It is a one-shot message system.

PART 3 — BACKEND / SOCKET EVENTS
==================================================

Add socket events or REST endpoints according to existing architecture.

Required events:

home:chat_send
payload:
{
  sessionId,
  senderPlayerId,
  receiverPlayerId,
  text,
  isAnonymous,
  destination
}

home:chat_receive
sent only to receiver phone for private messages

home:chat_inbox_update
sent to receiver phone after new private message or reaction

home:chat_reaction
payload:
{
  sessionId,
  messageId,
  playerId,
  emoji,
  textReply?: string
}

home:chat_mark_read
payload:
{
  sessionId,
  playerId,
  messageId
}

home:chat_toggle_dnd
payload:
{
  sessionId,
  playerId,
  dnd
}

home:chat_tv_queue_update
broadcast to TV when new TV message is queued

home:chat_tv_show_batch
TV uses this when showing up to 5 messages

Server rules:
- max text length 160 chars
- trim empty messages
- block duplicate consecutive same text from same sender to same target
- if destination private and receiver has 3 unread messages from same sender, reject:
  "L’utente ha già messaggi in attesa."
- if receiver DND is active:
  save message to inbox but do not send popup/badge
- destination tv:
  receiverPlayerId can be null or specific target for "message for X"
  message goes into tvMessageQueue, never appears immediately during active gameplay

PART 8 — TV MESSAGE QUEUE
==================================================

TV messages must NEVER interrupt gameplay.

They are shown only in safe windows.

Safe windows include:
- between games
- between rounds
- result screen
- ranking screen
- booking phase
- waiting phase
- karaoke transition between singers
- karaoke voting phase
- mini podium
- session idle

Unsafe windows include:
- active timer gameplay
- live quiz question
- active WordBack word
- active Ballo performance
- active Karaoke singer
- active Freestyle battle
- active Percorso mission
- any moment where the message could suggest or distract

Implement function:

isSafeToShowTvMessages(session, roundPayload, phase): boolean

When safe:
- if tvMessageQueue has messages:
  show batch overlay for max 5 seconds
  max 5 messages per batch
  mark them as shown
  remove from queue or move to shown list
- if queue empty:
  do nothing, no delay

If more than 5 messages:
keep remaining in queue for next safe window.

TV overlay:
Title:
💌 Messaggi dalla sala

Each message:
- if anonymous: “Anonimo”
- if signed: sender nickname
- if target exists: “per [receiverNickname]”
- text

Do not show during active gameplay.

==================================================
PART 9 — TV BATCH TIMING
==================================================

When safe window starts:
- show up to 5 TV messages
- total duration 5 seconds
- then disappear automatically
- game continues normally

If no messages:
- do not wait 5 seconds
- continue immediately

Important:
TV overlay must be non-blocking.
It should not change game phase.
It should not stop timers.
It should not require host action.

STOP adding features.

Fix this globally.

PART 2 — GLOBAL CLIENT SYNC PROBLEM
==================================================

Problem:
Some phones, especially Chrome, do not follow game state changes.
They remain stuck until manual page reload.

Manual reload works, so:
- backend/session state is probably correct
- client socket/polling/reducer/state replacement is unreliable

Required:
Make HomeJoin client self-healing.

==================================================
PART 3 — SERVER STATE VERSIONING
==================================================

Add monotonic state version to every Home session update.

Server:
Every time session state changes, increment:

homeStateVersion: number

Include it in every:
- home:state
- home:round
- polling GET /home/sessions/:id
- any game-specific state if possible

Examples of state-changing operations:
- game selected
- phase changed
- booking changed
- round changed
- voting opened/closed
- quiz question changed
- karaoke phase changed
- adult phase changed
- percorso phase changed
- coppie card state changed

Client stores:
lastSeenHomeStateVersion

If received version > local:
replace local session state.

If local version is stale for too long:
force refetch.

==================================================
PART 5 — FORCE RESYNC FUNCTION
==================================================

Implement one canonical function:

forceResync(reason)

It must:
1. GET current session from server
2. GET current players if needed
3. replace session state entirely
4. replace roundPayload entirely
5. re-identify current player from playerId
6. emit join:home
7. emit home:player_register
8. reset only transient local UI states:
   - overlays
   - pending clicks
   - local timers
   - local answered flags only if server phase changed
9. recompute timers from server timestamps
10. show small toast:
   “Riallineato”

Do not require browser reload.

Logs:
[HomeSync] forceResync start
[HomeSync] reason
[HomeSync] fetched version
[HomeSync] local version
[HomeSync] socket rejoined
[HomeSync] state replaced
[HomeSync] timer aligned

PART 7 — POLLING FALLBACK
==================================================

Add fallback polling every 5 seconds on HomeJoin while in active game.

Polling:
GET session state.

If returned homeStateVersion > local version:
- replace local state
- log:
  [HomeSync] polling caught stale state

If same version:
do nothing.

This is fallback only, not primary.

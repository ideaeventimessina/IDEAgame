FIRST AUDIT, THEN PATCH.
Do not touch Ballo.

Do not touch Quizzone.
Do not touch Coppie.
Do not touch audio engine unless adding optional existing stingers.
Do not redesign global Home Mode.

GOAL:
Transform "Percorso a Risate" into a simple, fast, phone-driven party game based on IMPROVISED MISSIONS.

Core idea:
- TV explains the mission clearly.
- Phones allow booking, voting, validation, and public interaction.
- No new hardware.
- No complex AI.
- Every mission must be understandable in 3 seconds.

GENERAL RULES:
- The game contains 10 mini-missions.
- Each mission has 2 main players unless mission-specific rule says otherwise.
- Players change every mission.
- Other connected players become PUBLIC ACTIVE.
- Public can vote, validate, choose prompts, or trigger choices from phones.
- Each completed mission gives 200 points.
- If player fails, the player or public member who caused/validated failure may gain 50 points when applicable.
- Voting missions use 1–5 stars.
- Each star is worth 20 points.
- Max vote score = 100 unless mission has fixed 200-point success.

COMMON FLOW FOR EACH MISSION:
1. TV shows mission title + one-line rule.
2. Booking phase: 2 players book.
3. Public interaction phase where needed:
   - vote prompt
   - choose object
   - choose song
   - choose yoga pose
   - choose tongue-twister
4. Mission starts.
5. TV becomes battlefield:
   - timer
   - mission instruction
   - selected prompt/image/song
   - public actions/validations
6. Public validates/votes from phones.
7. Server assigns points.
8. TV shows result.
9. Move to next mission with new booking.

IMPORTANT:
- Do not create 10 separate hardcoded games.
- Create a mission engine/config:
{
  missionId,
  title,
  type,
  playerCount,
  duration,
  publicActionType,
  scoringType,
  payload
}

MISSION ENGINE PHASES:
- mission_intro
- booking
- public_choice
- active
- voting
- result

PHONE ROLES:
- booked players: see their mission instructions.
- public players: see voting/validation/action UI.
- all phones must stay locked as app experience, no horizontal scroll.

TV:
- clear huge mission title
- clear role labels
- huge timer
- huge selected prompt/object/song/image
- visible public events
- result overlay

==================================================
MISSION 1 — IL GIORNALISTA / NON DIRE SÌ
==================================================

Players:
- Player A = concorrente
- Player B = giornalista

TV:
MISSIONE: NON DIRE "SÌ"
Regola: rispondi a 10 domande in 30 secondi senza dire "sì".
Errori: 0/2

Journalist phone:
Shows 10 fast questions designed to make the player answer “yes”.
One question at a time.
Button: "PROSSIMA DOMANDA".

Example questions:
- Ti stai divertendo?
- Vuoi vincere?
- Ti piace la pizza?
- Sei pronto?
- Hai mai mentito?
- Hai fame?
- Hai Instagram?
- Sei competitivo?
- Ti piace questa festa?
- Vuoi continuare?

Public phone:
Huge button:
"HA DETTO SÌ"

Validation:
If enough public validations arrive, count as one error.
Threshold:
- at least 2 public validations
OR 40% of connected public, whichever is smaller but minimum 1.

Rules:
- 2 errors = mission failed.
- If player survives 30s / 10 questions = player +200.
- If journalist makes player fail = journalist +100.
- If journalist fails = journalist -100, clamp score to 0 if negative scores not allowed.

SCORING RULES
==================================================

Default:
- mission completed = +200 to performing player/team
- public star vote = average stars * 20
- fail reward = +50 to validating/public-causing player when mission supports it

  player fails = +100 journalist
  journalist fails = -100 journalist

Clamp scores to 0 unless negative scores are explicitly supported.

PART 2 — SERVER-SOURCE TIMER RULE
==================================================

Implement/standardize one helper pattern:

Every timed phase must have:

{
  phaseStartedAt,
  phaseEndsAt
}

or phase-specific:
{
  questionStartedAt,
  questionEndsAt,
  votingStartedAt,
  votingEndsAt,
  missionStartedAt,
  missionEndsAt
}

Phones and TV calculate:

remaining = max(0, phaseEndsAt - Date.now())

Do NOT start independent local countdowns from full duration.

If a phase has no server timestamp:
add one.

==================================================
PART 3 — TIMER EXPIRY MUST ADVANCE STATE
==================================================

For every timed phase:

When Date.now() >= phaseEndsAt:
server must automatically close the phase.

Examples:
- Quiz question expired → reveal / time expired
- Sara’Musica expired → reveal / time expired
- Percorso active expired → voting or result
- Adult voting expired → close vote/result
- Karaoke voting expired → close vote/ranking/transition
- Coppie preview expired → hide cards
- WordBack expired → timeout
- Ballo expired → voting/podium

No phase may remain waiting with time=0.

Add guard:
if timeLeft <= 0 and phase is active:
force transition to next valid phase.

==================================================
PART 4 — LATE ACTION REJECTION
==================================================

Server must reject any action submitted after phaseEndsAt:

- answers
- votes
- public choices
- mission validations
- star ratings
- taboo alarms if round closed
- visibility requests if timer expired

Return:
409 time_expired

Phone:
show:
“Tempo scaduto”

Do not assign points.
